提交代码时忽略特定文件

Navigator 里面包含了项目所有的文件也会当成未提交的文件，里面包含了许多Eclipse为了管理项目的项目配置文件，对于我们来说是不需要的我们需要忽略这些特殊的文件

例如：

  • .classespath文件
  • .project文件
  • .settings文件

这些都是eclipse为了管理我们项目所创建的文件，我们并不需要

为什么要忽略这些文件：

每个人开发用的工具不同所以配置信息不同

每个开发工具有不同的配置信息如果不忽略掉当下载时会出现冲突

为了解决冲突我们必须忽略掉这些文件

关于java忽略示例文件

Java.gitignore
# Compiled class file
*.class

# Log file
*.log

# BlueJ files
*.ctxt

# Mobile Tools for Java (J2ME)
.mtj.tmp/

# Package Files #
.jar
.war
*.nar
*.ear
*.zip
*.tar.gz
*.rar

# virtual machine crash logs, see http://www.java.com/en/download/help/error_hotspot.xml
hs_err_pid*

在~/.gitconfig 文件中引入上述文件

[core]

​ excludesfile=C:/Users/Admin/Java.gitignore（这里这个路径是可以换的不是固定的而且这里的斜线一定是/）

操作步骤：

  • 创建一个以.gitignore结尾的文件，在里面可以指定以什么结尾的文件可以使用通配符*

  • 在全局配置文件.gitconfig添加配置

    [core]

    excludesfile=文件路径

  • 重新查看eclipse里面的文件带问号代表需要提交的，会发现navigator指定的文件上面已经没有标问号了，表示忽略了。

附上：关于eclipse中符号的意义

表示项目已具备初始化为git仓库 —–> 会有黄色圆柱形的符号 （提交后的文件也会变为这个符号）

表示以及添加到暂存区 —–> 文件夹上面标有* 文件上面标有+号

表示新添加的文件 ——> 文件上面标有？号 (此时需要添加到暂存区然后再提交)

如果在不同的本地库中我们进行修改而且修改的是同一行那么就会出现冲突

修改同一行代码之后分别进行提交：注意在提交的时候我们和SVN update一样需要更新一下需要执行pull操作拉取并合并
